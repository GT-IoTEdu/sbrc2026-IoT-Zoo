# 🛠️ PCAP to CSV Converter

This tool is part of the IoT-Zoo framework. It automates the extraction of network and application features from PCAP files generated by the IoT-Zoo testbed. It uses a synchronized fusion of Scapy (Network Layer) and Tshark (Application Layer) to create a high-dimensional dataset ready for Machine Learning.

## 📋 Prerequisites

Before running the converter, ensure you have the following dependencies installed on your host:

```bash
# Install Tshark
sudo apt-get update && sudo apt-get install -y tshark

# Install Python dependencies
pip install pandas scapy
```

## Usage
The ```main.py``` script orchestrates both Tshark and Scapy to generate a single enriched dataset.

```python
# Run the orchestrator
python3 main.py --input capture.pcap --output dataset_final.csv
```

## Dataset Features

The converter extracts 16 key features, providing a high-dimensional view for Machine Learning tasks:

| Feature | Source | Description |
|---------|--------|-------------|
| pkt_index | Tshark | Unique packet identifier (`frame.number`). |
| time_epoch | Tshark | High-precision arrival timestamp. |
| frame_len | Tshark | Total Ethernet frame length in bytes. |
| ip_src / ip_dst | Tshark | IPv4 source and destination addresses. |
| ip_proto | Tshark | L3 Protocol ID (e.g., `6` for TCP, `17` for UDP). |
| ip_ttl | Scapy | Time to Live (useful for spoofing detection). |
| src_port / dst_port | Tshark | L4 source and destination ports. |
| tcp_flags_hex | Tshark | TCP flags in hexadecimal format (e.g., `0x018`). |
| tcp_flags_str | Scapy | TCP flags in string format (e.g., `PA`, `S`, `A`). |
| tcp_seq | Scapy | TCP sequence number for session tracking. |
| app_protocol | Tshark | L7 Protocol (MQTT, RTSP, NTP, DNS, etc.). |
| mqtt_topic | Tshark | MQTT topic namespace (e.g., `city/air/co`). |
| mqtt_msgtype | Tshark | MQTT Message Type (Publish, Subscribe, etc.). |
| mqtt_qos | Tshark | Quality of Service level for the MQTT message. |
| mqtt_len | Tshark | MQTT application payload length. |

## 💡 Why Fused Extraction?
Traditional tools often drop "orphan" telemetry packets (unidirectional traffic). This converter preserves every valid IP packet, ensuring that the heterogeneity and temporal dynamics of the 43+ IoT-Zoo profiles are fully captured in the final CSV.
